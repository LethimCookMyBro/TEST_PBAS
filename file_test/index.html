🎯 เก็งข้อสอบ 100% (จากโพย)

🔥 เฉลยตรงเผงจาก "โพยข้อสอบ"

จากรูปโพย นี่คือคำตอบแบบกระชับ รวบยอด เพื่อจดเข้าห้องสอบ:

  • Hub vs Switch ต่างกันยังไง:
    Hub (Layer 1): ทำงานแบบ Broadcast ส่งข้อมูลออกทุกพอร์ต (เสี่ยงชนกัน Collisions/Half-Duplex)
    Switch (Layer 2): จำ IP/MAC Address ส่งข้อมูลหาปลายทางได้ตรงเครื่อง (Unicast/Full-Duplex ทำงานไวกว่า)
  • การทำ Broadcast CLI:
    ไม่มีคำสั่งตั้งค่า Broadcast ตรงๆ แต่มักจะหมายถึง Broadcast Storm ที่แก้ด้วย spanning-tree vlan X หรือถ้าบอกว่า "Broadcast Domain" คือการที่ Router/VLAN กั้น Domain ไว้ไม่ให้ข้อมูลทะลักไปวงอื่น
  • Encapsulation CLI:
    คำสั่งตอนทำ InterVLAN ฝั่ง Router (ROAS) คือ encapsulation dot1Q [vlan_id] เพื่อระบุว่า Sub-interface นี้ทำงานกับ VLAN หมายเลขอะไร
  • Route CLI:
    คำสั่งเปิดการ Routing บน L3 Switch คือ ip routing (สำคัญมาก ขาดไป PING ไม่เจอ) หากเป็น Router ทั่วไปคือพวก ip route 0.0.0.0 0.0.0.0 [next-hop]
  • InterVLAN Route concept (ลักษณะ 2 แบบ):
    1. Router-on-a-Stick (ROAS): ใช้ Router + Switch ต่อสาย Trunk 1 เส้น โหลดหนักที่ Router เป็นคอขวด
    2. Switch Virtual Interface (SVI): ใช้ L3 Switch ตัวเดียว ตั้งค่า IP ที่คำสั่ง interface vlan ประสิทธิภาพสูงกว่า
  • Protocol Spanning-Tree (cost etc. 4, 19):
    ค่า Cost สำหรับหา Root Port:
    - พอร์ต 1 Gbps (GigabitEthernet) = Cost 4
    - พอร์ต 100 Mbps (FastEthernet) = Cost 19
  • Etherchannel ประโยชน์ / การทำ 2 แบบ:
    ประโยชน์: รวมสาย Lan เพิ่ม Bandwidth และทำ Redundancy (เส้นนึงขาด อีกเส้นวิ่งแทน ไม่เน็ตหลุด)
    การทำ 2 แบบ (Protocol):
    - PAgP (ของ Cisco ผูกขาด) ใช้โหมด: desirable หรือ auto
    - LACP (ของ IEEE กลาง) ใช้โหมด: active หรือ passive
  • วิธีเขียน IPv6:
    กฎ 3 ข้อ: 1. ตัดเลข 0 ข้างหน้าทิ้ง (0202 -> 202) / 2. 0000 เขียนย่อเป็น 0 / 3. กลุ่ม 0 ที่ติดกันยาวๆ ยุบเป็น :: (แต่ทำได้แค่ 1 ครั้งต่อ IP ป้องกันความสับสน)
  • ACL รูปแบบ ลำดับ (deny any latest):
    ทำงานแบบ Top-Down (เช็คทีละบรรทัดจากบนลงล่าง) กฎตายตัว: บรรทัดสุดท้ายของ ACL จะมี "Implicit Deny Any" ซ่อนอยู่เพื่อบล็อกทุกอย่าง แปลว่าบรรทัดสุดท้าย(latest) ก่อนจบ เรามักต้องใส่ permit ip any any เพื่ออนุญาตคนที่เหลือเสมอ

CH9: InterVLAN Routing & L3 Switch

Summary

InterVLAN Routing

คือการทำให้ Host ที่อยู่ ต่าง VLAN กัน ติดต่อกันได้ โดยต้องมีอุปกรณ์ Layer 3 มาช่วย

วิธีที่ 1: Router-on-a-Stick (ROAS)

  • ใช้ Router ต่อกับ Switch ผ่าน Trunk link 1 เส้น
  • Router มี Sub-interfaces แต่ละตัวแทน VLAN
  • ข้อมูลถูกส่งไป Router แล้ว Route กลับไปยัง VLAN ปลายทาง

Config Router (ROAS)

Router(config)# interface GigabitEthernet 0/0/0.10 Router(config-if)# encapsulation dot1q 10 Router(config-if)# ip address 192.168.10.1 255.255.255.0 Router(config)# interface GigabitEthernet 0/0/0.20 Router(config-if)# encapsulation dot1q 20 Router(config-if)# ip address 192.168.20.1 255.255.255.0

การทำงาน ROAS

  1. 192.168.10.10 ส่งข้อมูลให้ 192.168.20.10
  2. Router รับเฟรม Tag VLAN 10 บน sub-interface
  3. Router ถอด Frame Header หาเส้นทางไปวง 192.168.20.0/24
  4. ส่งไป sub-interface ที่ติด Tag VLAN 20 กลับไป Switch ผ่าน Trunk

วิธีที่ 2: Switch Virtual Interface (SVI)

  • ใช้ L3 Switch ที่รองรับ Routing โดยไม่ต้องใช้ Router เพิ่ม
  • SVI = Virtual Interface ของแต่ละ VLAN
  • ใช้คำสั่ง ip routing เปิดใช้งาน

Config L3 Switch (SVI)

Switch(config)# vlan 10 Switch(config)# exit Switch(config)# vlan 20 Switch(config)# exit Switch(config)# interface vlan 10 Switch(config-if)# ip address 192.168.10.1 255.255.255.0 Switch(config)# interface vlan 20 Switch(config-if)# ip address 192.168.20.1 255.255.255.0 Switch(config)# ip routing Switch(config)# interface gigabit 0/1 Switch(config-if)# switchport trunk encapsulation dot1q Switch(config-if)# switchport mode trunk

ROAS vs SVI

หัวข้อROASSVI
อุปกรณ์Router + SwitchL3 Switch ตัวเดียว
คำสั่งencapsulation dot1qinterface vlan + ip routing
ประสิทธิภาพต่ำกว่า (คอขวดที่ Router)สูงกว่า

CH10: Spanning Tree Protocol (STP)

Summary

ปัญหาของ Network Redundancy

  1. Broadcast Storm - Switch ส่ง broadcast ต่อกันไม่สิ้นสุด เครือข่ายล่ม
  2. Multiple Frame Transmission - เครื่องปลายทางรับ Frame ซ้ำ เสีย bandwidth
  3. MAC Database Instability - MAC table สับสน ส่งกลับไม่ถูก

STP (Spanning Tree Protocol)

โปรโตคอล Layer 2 ป้องกัน Loop ในเครือข่าย Ethernet

หลักการทำงาน

  1. เลือก Root Bridge
  2. คำนวณเส้นทางดีสุด (Root Path Cost)
  3. เลือก Port ที่จะ Block
  4. ส่ง BPDU

BPDU (Bridge Protocol Data Units)

เฟรมพิเศษที่ส่งระหว่าง Switch

  • Root Bridge ID
  • Cost to Root Bridge
  • Sender Bridge ID
  • Port ID
  • STP timers

การเลือก Root Bridge

  • Bridge ID = Priority + MAC Address
  • ค่าเริ่มต้น Priority = 32768
  • ช่วง: 0-61440 (หาร 4096 ลงตัว)
  • Bridge ID ต่ำสุด = Root Bridge
  • Priority เท่ากัน → เทียบ MAC Address
VLAN 1: 32768 + 1 = 32769 VLAN 10: 32768 + 10 = 32778 VLAN 100: 32768 + 100 = 32868

STP Port Costs

ความเร็วCost
10 Gbps2
1 Gbps4
100 Mbps (FastEthernet)19
10 Mbps100

ประเภท Port

Portคำอธิบาย
Root Port (RP)เส้นทางดีสุดไป Root Bridge (ทุก Non-Root Switch มี 1 RP)
Designated Port (DP)ตัวแทนในแต่ละ segment รับ-ส่ง frame (ไม่ซ้ำ RP)
Non-Designated (NDP)ถูก Block เพื่อหยุด Loop (cost มากสุด)

STP Process & Timers

Normal (เสียบสายใหม่)

  1. BLOCKING
  2. LISTENING (15s) - ส่ง/รับ BPDU
  3. LEARNING (15s) - เรียนรู้ MAC
  4. FORWARDING - ส่งข้อมูลได้

Switch Link Down

  1. BLOCKING (20s)
  2. LISTENING (15s)
  3. LEARNING (15s)
  4. FORWARDING

รวม = 20+15+15 = 50 วินาที

คำสั่ง STP

! ปิด STP Switch(config)# no spanning-tree vlan 1 ! เปิด STP Switch(config)# spanning-tree vlan 1 ! ดูสถานะ Switch# show spanning-tree

CH10 (ACL): Access Control Lists

Summary

ACL (Access Control List)

กลไกควบคุมการเข้าถึงเครือข่ายโดยกรองแพ็กเก็ตที่เข้า/ออกจาก Router

ประเภท ACL

ประเภทหมายเลขกรองตาม
Standard1-99, 1300-1999Source IP เท่านั้น
Extended100-199, 2000-2699Source/Dest IP, Protocol, Port

รูปแบบ

แบบตัวอย่าง
NumberStandard 1-99, Extended 100-199
Namedใช้ชื่อแทนตัวเลข

ขั้นตอนตั้งค่า ACL

  1. สร้าง ACL
  2. ประกาศใช้ ACL (ผูกกับ Interface)

Standard ACL Named

! สร้าง ACL Router(config)# ip access-list standard TEST Router(config-std-nacl)# permit host 10.0.0.10 ! ประกาศใช้ ACL Router(config)# interface F0/0 Router(config-if)# ip access-group TEST in ! หรือ Router(config-if)# ip access-group TEST out

Wildcard Mask

คำนวณ: 255.255.255.255 - Subnet Mask

Subnet MaskWildcardความหมาย
255.255.255.2550.0.0.0ตรง host เดียว
255.255.255.00.0.0.255Network /24
255.255.0.00.0.255.255Network /16
0.0.0.0255.255.255.255ทุก IP (any)

0 = ต้องตรง | 255 = ไม่สนใจ

การเลือกจุดใช้ ACL

  • Standard ACL วางใกล้ Destination (ปลายทาง)
  • Extended ACL วางใกล้ Source (ต้นทาง)
  • in = กรองแพ็กเก็ตที่เข้า interface
  • out = กรองแพ็กเก็ตที่ออก interface

Extended ACL

Router(config)# ip access-list extended [NAME] Router(config-ext-nacl)# [permit/deny] [protocol] [source] [wildcard] [dest] [wildcard] eq [port] ! ตัวอย่าง: บล็อก 10.0.0.10 ไม่ให้เข้าเว็บ 192.168.10.10 access-list 100 deny tcp host 10.0.0.10 host 192.168.10.10 eq 80 access-list 100 permit ip any any

ท้าย ACL มี implicit deny any เสมอ - ต้องใส่ permit ip any any

CH11: EtherChannel

Summary

EtherChannel คืออะไร?

รวม link หลายเส้นเข้าด้วยกัน เพิ่ม bandwidth+ความทนทาน สูงสุด 8 พอร์ต/channel

  • Bandwidth: 2x1Gbps = 2Gbps, 4x1Gbps = 4Gbps
  • Redundancy: link หนึ่งล้ม ที่เหลือยังทำงาน
  • ลดภาระ STP: มองเป็น link เดียว

ข้อจำกัด

  • Port ต้องความเร็วเท่ากัน
  • Port ต้องโหมดเดียวกัน
  • สูงสุด 8 links/channel
  • ต้องรวมทั้งสองฝั่ง

โหมด EtherChannel

โหมดProtocolคำอธิบายใช้ร่วมกับ
onManualกำหนดเอง ไม่เจรจาon
desirablePAgP (Cisco)เริ่มร้องขอauto, desirable
autoPAgP (Cisco)รอฝั่งตรงข้ามdesirable
activeLACP (IEEE)เริ่มร้องขอpassive, active
passiveLACP (IEEE)รอฝั่งตรงข้ามactive

PAgP = Cisco | LACP = IEEE 802.3ad ทุกยี่ห้อ

คำสั่ง Config

Static (on)

Switch(config)# interface range Gi1/0/1 - 2 Switch(config-if-range)# channel-group 1 mode on

PAgP

Switch(config-if-range)# channel-group 1 mode desirable ! หรือ mode auto

LACP

Switch(config-if-range)# channel-group 1 mode active ! หรือ mode passive

ดูการตั้งค่า

Switch# show etherchannel summary

CH13: IPv6

Summary

IPv6 คืออะไร?

  • IPv4 = 32-bit ~ 4.3 พันล้าน
  • IPv6 = 128-bit ~ 3.4 x 1038 addresses

IPv4 vs IPv6

หัวข้อIPv4IPv6
Address32-bit128-bit
Header Fields137 (เร็วขึ้น)
Header Sizeไม่แน่นอนคงที่ 40 bytes
Checksumมีไม่มี
Optionsใน HeaderExtended Header
Auto ConfigDHCPStateless+DHCP
SecurityIPsec optionalIPsec mandated
Broadcastมีไม่มี
Min MTU576B1280B
FragmentRouter ระหว่างทางSource เท่านั้น
TTLTime-To-Live (วินาที)Hop Limit (hops)

การเขียน IPv6

8 กลุ่ม x 16-bit เขียนเลขฐาน 16 คั่นด้วย :

กฎย่อ

  1. ตัดเลข 0 นำหน้า: 0dcb → dcb
  2. 0000 เขียนเป็น 0
  3. กลุ่ม 0000 ติดกัน แทนด้วย :: (ทำได้ 1 ครั้ง)

ตัวอย่าง

เต็มย่อ
fe80:0000:0000:0000:0202:b3ff:fe1e:8329fe80::202:b3ff:fe1e:8329
2001:0000:0000:34fe:0000:0000:00ff:03212001::34fe:0:0:ff:321
0000:...:0001::1

ประเภท IPv6 Address

ประเภทรูปแบบคำอธิบาย
Unicastone-to-oneเครื่องหนึ่งไปอีกเครื่อง
Multicastone-to-manyส่งไปกลุ่ม Group
Anycastone-to-one-of-manyส่งไปเครื่องที่ใกล้สุดใน set

IPv6 ไม่มี Broadcast!

Unicast Types

ชนิดPrefixเทียบ IPv4
Global Unicast2000::/3Public IP
Link-LocalFE80::/10169.254.x.x
Unique LocalFC00::/7Private IP

Extended Header

  1. Hop-by-Hop - ทุก Router ต้องอ่าน (Jumbogram max 232B)
  2. Destination Options
  3. Routing - Source Routing (Strict/Loose)
  4. Fragment - ที่ Source เท่านั้น (PMTUD)
  5. Authentication - IPsec
  6. ESP - Encryption

Backward Compatibility

  • Dual Stack - ใช้ IPv4+IPv6 พร้อมกัน
  • Tunneling - ห่อ IPv6 ด้วย IPv4 Header
  • Translation - แปลง IPv6↔IPv4
  • IPv4-mapped: 222.1.41.90 → ::FFFF:222.1.41.90
